ＦＴオプション【光通信プログラム】DOC


 　ファイル・トランシーバー（ＦＴ）用オプションプログラム
  　　　　光通信プログラム  Version 0.3                    制作者名 ： たろサ

【1】 紹介
　　　このプログラムはＦＴのオプションプログラムです。ＦＴが無いと実行が
    できません。

 　　　・ザウルスからパソコンへのファイルの送信
 　　　・パソコンからザウルスへのファイルの個別受信
 　　　・パソコンからザウルスへのファイルの一括受信
 　　　・パソコンからザウルスへのアドインのダウンロード

　　　ができます。

　　　　また、このドキュメントに示す解説、解析結果、及び本プログラムは、
　　　ザウルスの外部動作から独自に類推して作成されたもので、完全である保
　　　証はありません。最悪の場合は、本プログラムの影響でザウルスのデータ
　　　等を破壊してしまう恐れもあります。さらに、このプログラムでは、CHAIN
　　　等のコマンドを使用しているという点で、アドインソフト開発規制に違反
　　　しております。利用される方は、くれぐれも注意して（データのバックア
　　　ップを取る等）、使用は各人の責任において行って下さい。


【2】 動作環境
　　　　このプログラムはザウルスＰＩ−６０００、ＰＩ−５０００、ＰＩ−
　　　４５００上で動作確認をしております。
　　　　さらに、ザウルスのアドインソフトＦＴが必要です。

【3】 使用法
　　　　先ず最初に、アドインダウンローダー等を用いて、ザウルスにダウンロ
　　　ードして下さい。ただし、単独での実行はできません。ＦＴのメニューに
　　　登録され、ＦＴから実行します。

　　　　ＦＴのメニューに登録する方法は、Ａｄｄ　ｉｎのメニュー画面から、
　　　一度、本プログラムを実行して下さい。自動的に、ZFT.BA0 を書き換えて、
　　　ＦＴのメニューに登録されます。

　【 送　信 】
　　　　　ＦＴで【光通信】→【送信】を選択すると、本プログラムが起動し、
　　　　画面に送信ファイル名が表示されます。
　　　　　受け側の準備ができたことを確認して、【開始】タッチして下さい。
　　　　具体的な手順は以下に示します。

　　　　　光通信には、パソコン側で光通信ツールが必要です。

　　　　　私は、おやＰ氏が作られた「ZAURUS 光通信支援ツールｏｃｐｚ」を
　　　　使っています。おやＰさん、このようなプログラムを公開したいただき、
　　　　どうもありがとう御座いました。この場を借りてお礼申し上げます。

　　★尚、ファイル通信には、ｏｃｐｚ ver1.03 以上が必要です。
                                     ^^^^^^^
　　　　　ザウルスが「受け側の準備をして下さい」と表示したら、パソコンで
　　　　 OCPZ を起動して下さい。私は、以下のようなバッチファイルで起動し
　　　　ています。

　　　　　　set ocpzwait=100;5000
　　　　　　speed r0 9600 b8 po s1 none
　　　　　　ocpz aux %1 -b

　　　　　ｏｃｐｚの「ファイル転送開始」の文字を確認したら、ザウルスの
　　　　【開始】をタッチして下さい。アップロードが始まります。

　　　　　終了すると、「ファイルの送信が終了しました。パソコンの受信を終
　　　　了させて下さい。」というメッセージがでるので、ｏｃｐｚを終了させ
　　　　て下さい。

　　★おやＰ氏作のｏｃｐｚ は NIFTY-SREVE FENOTE LIB 2 にあります。

　　★複数ファイルの連続送信方法
　　　　　裏わざとして、選択した複数ファイルを一度に送信する方法がありま
　　　　す。
　　　　　ＦＴ本体の機能である【ザウルス通信】→【送信】を選択して、その
　　　　送信データをパソコンで受けてやればＯＫです。どんなファイルが送ら
　　　　れてくるかは、ヘッダファイル（★表１参照）として、最初に送られて
　　　　くるので、受け側でこれに対応して受信すればＯＫです。

　★表１．【ヘッダファイルの内容(例)】
    (ABCD.BAS と ABCD.BA0 の２つのファイルを送る場合)
     2[CR][LF]              →　ファイル数
    ABCD.BAS[CR][LF]        →　ファイル名
    ABCD.BA0[CR][LF]        →　ファイル名
    [EOF]

　　　[CR] は、0Dh コード、[LF] は、0Ah コード、[EOF] は、1Ah コード。



　【個別受信】
　　　　　受信には、個別受信と一括受信があります。

　　　　　ＦＴで【光通信】→【個別受信】を選択すると、本プログラムが起動
　　　　し、ﾌｧｲﾙ名= と表示して、受信ファイル名の入力待ちとなります。

　　　　　ファイル名を入力すると、受信中になり、パソコンからのファイル転
　　　　送を待ってます。パソコンからファイルを転送して下さい。
　　　　　受信が終了すると、再び、ファイル名待ちになります。ここで、もう
　　　　受信を終了したければ、何も入力しないで、【入力】キーだけを押して
　　　　下さい。
　　　　　個別受信が終了します。


　【一括受信】（アドインのダウンロードに使えます）
　　　　　受信には、一括受信と個別受信があります。

　　　　　ＦＴで【光通信】→【一括受信】を選択すると、本プログラムが起動
　　　　し、受信待ちになります。【開始】で受信を開始します。

　　　　　開始すると、ファイルの名前が入ったヘッダファイルを待っています。
　　　　ザウルスのアドインを転送する時は、ＺＲＳファイルを待っています。

　　★普通のファイルのダウンロード
　　　　　ヘッダファイルとして、転送するファイル数、転送ファイル名を書いた
　　　　ファイルを最初に送って下さい。書式は★表１と同じですので、参照して
　　　　下さい。（★表１参照）

　　★ザウルスのアドインのダウンロード
　　　　　送信側（パソコン）は、ＺＲＳファイルを送って、その後、順次にＢＡ
　　　　Ｓファイルを送ればアドインのダウンロードができます。

　　　　　例えば、ＡＢＣＤというプログラムをダウンロードするとします。ダウ
　　　　ンロードファイルはＡＢＣＤ．ＢＡＳとＡＢＣＤ．ＢＡ０とします。これ
　　　　をダウンロードするには、ｏｃｐｚで、ＡＢＣＤ．ＺＲＳ、ＡＢＣＤ．Ｂ
　　　　ＡＳ、ＡＢＣＤ．ＢＡ０を順番に送ります。

         >ocpz -b abcd.zrs aux
         >ocpz -b abcd.bas aux
         >ocpz -b abcd.ba0 aux

　　　　　プログラムファイルがＢＡ０，ＢＡ１，ＢＡ２・・・・とたくさんある
　　　　場合は、続けて送れば大丈夫です。

　　★参考にｏｃｐｚ を使ったバッチファイルの例を書いておきます。
　　　これは、引数にプログラムファイル名と、拡張子を付ければアドインを転送
　　　してくれます。
　　　　ＩＦ文がづらづらならんでいるのは、間違ってＤＯＣを送ったり、引数に
　　　ＺＲＳを書いたりしたときのためのチェックルーチンです。

------- ADDIN2Z.BAT --------
             @ECHO Off
             IF NOT EXIST %1.ZRS GOTO :EXIT
             SET ZFT=%1
             SPEED R0 9600 B8 PO S1 NONE
             ocpz -b %ZFT%.ZRS aux
             :LOOP
             SHIFT
             IF "%1"=="" GOTO EXIT
             IF "%1"=="doc" GOTO LOOP
             IF "%1"=="doC" GOTO LOOP
             IF "%1"=="dOc" GOTO LOOP
             IF "%1"=="dOC" GOTO LOOP
             IF "%1"=="Doc" GOTO LOOP
             IF "%1"=="DoC" GOTO LOOP
             IF "%1"=="DOc" GOTO LOOP
             IF "%1"=="DOC" GOTO LOOP
             IF "%1"=="zrs" GOTO LOOP
             IF "%1"=="zrS" GOTO LOOP
             IF "%1"=="zRs" GOTO LOOP
             IF "%1"=="zRS" GOTO LOOP
             IF "%1"=="Zrs" GOTO LOOP
             IF "%1"=="ZrS" GOTO LOOP
             IF "%1"=="ZRs" GOTO LOOP
             IF "%1"=="ZRS" GOTO LOOP
             IF EXIST %ZFT%.%1 ocpz -b %ZFT%.%1 aux
             PAUSE
             GOTO LOOP
             :EXIT
             SET ZFT=
------- ここまで --------

使用例：> ADDIN2Z ABCD BAS BA0

　　　　　ＡＢＣＤというアドインの ABCD.BAS ABCD.BA0 を送るので、引数にＡ
　　　　ＢＣＤ　ＢＡＳ　ＢＡ０の３個が付いてます。

　　★ＺＲＳファイルとは、アドインローダー（jsfree1.exe）がアドインを転送
　　　するときに読み込むファイルのことです。

　　　・１行目にＺＲＳ，Ｐ，”タイトル名”が書かれています。
　　　・２行目はドキュメントファイル名が書かれています。
　　　・３行目以降は、転送ファイル名が順番に書かれています。

　　　　　以下にＺＲＳファイルを参考に書いておきます。
------- ZFT.ZRS --------
             ZRS,P,"ファイル トランシーバー Ver x.xx by たろサ"
             ZFT.DOC
             ZFT.BAS
------- ここまで --------

　　　　　本プログラムでは、ＺＲＳファイルのＺＲＳ，Ｐ，”・・・”，ＺＦＴ
　　　　．ＤＯＣの４つのデータを読み飛ばしてＺＦＴ．ＢＡＳからファイルを作
　　　　成して受信受け付けを行います。

　　　　　また、この方法でアドインをダウンロードしますと、ＢＡＳファイルが
　　　　中間コードで無くても送信する事ができます。
