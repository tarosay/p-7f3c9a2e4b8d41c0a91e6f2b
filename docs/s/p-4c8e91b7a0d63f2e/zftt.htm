【１】オプションプログラムの作り方

　１−１ オプションプログラムの仕組み
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
　　　　　ＦＴは、選択したファイル名とメインメニュー、サブメニューの何を
　　　　選んだかを ZFT.BA1 というファイルに残して、変数Ａ＄以外のものを解
　　　　放して、オプションプログラムをＣＨＡＩＮして呼び出します。Ａ＄に
　　　　はＣＨＡＩＮするオプションプログラムのファイル名が格納されていま
　　　　す。

　　　　　ＦＴ用オプションプログラムは ZFT.BA1 の内容を読んで、プログラム
　　　　が走ります。
　　　　　オプションプログラムの終了時に、必ず CHAIN "SX:ZFT.BAS" を実行
　　　　するようにして下さい。そうするとＦＴが立ち上がり、終了と同時にＦ
　　　　Ｔに帰ってこれます。
　　　　　ＦＴの起動時に変数等はクリアしますので、オプションプログラムで
　　　　は、自由に変数を用いても大丈夫です。

　　　　　また、ＦＴはオプションプログラムを起動するとき以外は、ZFT.BA1を
　　　　消去するようにしています。ZFT.BA1ファイルが存在するかどうかで、オ
　　　　プションプログラムが、ＦＴのメニューから起動されたものなのか、単
　　　　独で起動されたものなのかを判断できます。

　　★参考資料として、オプションプログラムのソースデータを同梱して置きます。


　１−２ オプションプログラムのメニュー登録
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
　　　　　オプションプログラムメニューは ZFT.BA0 ファイルに書かれています。

　下記に、その一例を示します。
------- ZFT.BA0 --------
TUHI,光 通 信,送　　信,一括受信,個別受信,@
TUOP,Ｏption１５,アップロード,@
ZA2B,ＢＡＳＩＣ変換,中間コード化,アスキーコード化,@
ZGRW,画像ビューア,@
[EOF]
------- ここまで --------
[EOF] は、1Ah コード。（ファイル末コードは必要です。）

　　　・データはカンマで区切って、行末に @ を入れて下さい。
　　　・先頭は、ファイル名です。ファイル名は４文字固定です。
　　　・２番目は、メインタイトルです。
　　　・３番目以降は、サブタイトルです。
　　　・サブタイトルは、９個までです。

　　★ＦＴはオプションプログラムのファイル名を検索して、存在すれば、メ
　　　ニューに登録します。
　　★上記のように、オプションプログラムのファイル名は、４文字固定です。
　　　すいませんが、仕様ですので、御願いします。

　　　　ここで、ZFT.BA0 をどうにかして、作らなくてはいけません。パソコン
　　　で、ファイルを作って、ザウルスに転送してもよいですが、面倒なので、
　　　オプションプログラム側で、自動的にZFT.BA0 を検索して、自分自身がメ
　　　ニューに存在しなければ、自動登録するようにしました。
　　　　　　　　　　　　．．．．．
　　　　このアイデアは、須賀浦 敦さんのアイデアを流用させて頂きました。ど
　　　うも、ありがとう御座いました。この場を借りまして、お礼申し上げます。

　　　アルゴリズムとしては、
　　　　1.ZFT.BAS が存在しない　→　終了へ
　　　　2.ZFT.BA1 が存在する　　→　6 へ・・・
　　　　3.ZFT.BA0 が存在しない　→　ZFT.BA0 をOUTPUTでオープン、書込 → 終了へ
　　　　4.ZFT.BA0 が存在する　　→　ZFT.BA0 をAPPENDでオープン、書込 → 終了へ
　　　　5.ZFT.BA1 の１行目が自分のファイル名でない　→　終了へ
　　　　6.本来の処理の開始・・・

　　　というような感じです。

　１−２−１ ＦＴにメニュー登録ルーチン内蔵
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
　　　　さらに、Ｖｅｒ０．９８以降から、ＦＴ本体にこれらのオプションのメ
　　　ニュー登録プログラムを内蔵しました。

　　　　オプションプログラムでは、ZFT.BA1ファイルの存在を確かめて、存在し
　　　なければ、メニューに追加したい行を１行、ZFT.BA1ファイルに書いて、Ｆ
　　　Ｔの*ZFTCHのラベルにＣＨＡＩＮして下さい。
　　　　以下に画像ビューアの例を書きます。（Ａ）がZFT.BA1ファイルの存在確
　　　認です。存在しなければ、（Ｂ）に飛びます。（Ｂ）では、メニューデータ
　　　をZFT.BA1ファイルに書いてＦＴをＣＨＡＩＮしています。

　【注意】
　★ファイルのオープンはかならず、バイナリモード”,B”でオープンして下さい。
　★ＰＲＩＮＴ　＃１命令の最後にセミコロン；を付けるのを忘れないようにし
　　て下さい。

　（Ａ）ON ERROR GOTO *FTCH:OPEN "SX:ZFT.BA1"FOR INPUT AS #1:CLOSE #1

　（Ｂ）*FTCH:OPEN"SX:ZFT.BA1"FOR OUTPUT AS #1,B:PRINT #1,"ZGRW,画像ビューア,@";:CLOSE #1:CLEAR:ON ERROR GOTO 0:CHAIN "SX:ZFT.BAS",*ZFTCH


　１−３ オプションプログラム用データのＢＡ１ファイルの説明
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
　　　　ＦＴが残すＺＦＴ．ＢＡ１について説明します。

　　　・先頭行は、どのファイルの何番目のサブメニューを選んだかが書かれて
　　　　います。
　　　　　ファイル名の後ろに付いて書いてある数字がメニュー番号です。１は
　　　　１番目のサブメニューを意味します。もし、サブメニューが無い場合は、
　　　　０が付きます。
　　　・２行目は選択ファイルの数が書いてます。もし、選択ファイルが無い場
　　　　合は、０となります。
　　　・３行目以降はファイル名が順に入ってます。もし、選択ファイル数が０
　　　　の場合は、ここの行は存在しません。

------- ZFT.BA1 --------
ZA2B1
2
ABCD.BAS
ABCD.TXT
[EOF]
------- ここまで --------


　１−４ オプションプログラムのファイル名について
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
　　　　オプションプログラムのファイル名は、先述したように、４文字固定で
　　　す。
　　　　さらに、先頭の文字が R で始まっているファイルについて、ＦＴの方で
　　　特殊な意味を持たせました。

　　R***.BAS となるオプションプログラムをＦＴがＣＨＡＩＮして実行するとき
　　ＦＴの方で選択したファイルが実際にＳＸドライブに存在するかどうかをチェ
　　ックしないようにしました。

　　　これは、パソコンリンクを使うために付けた仕様です。ＦＴがファイル名
　　の保存用に用いているZFT.BA3 というファイルをリンクしているザウルスの
　　ファイル内容に置き換えてＦＴから操作させようという考えに基づいていま
　　す。このアイデアは'須賀浦 敦'さんのアイデアです。すばらしいアイデアを
　　提供していただいて、ありがとう御座いました。この場を借りて、お礼申し
　　上げます。

　　　★★　みなさん、頑張って、ＦＴをファイルマネジャーに使った　★★
　　　★★　オプションプログラムを組んで下さい。　　　　　　　　　★★


【２】ザウルス−ザウルス間通信の説明

　　　　ザウルス同士の送受信方法について説明します。

　　　　最初にヘッダファイルを送信して、受け側のザウルスに、転送ファイ
　　　ル数とファイル名を送った後、順番にファイルを転送していきます。
　　　　ヘッダのフォーマットは以下の表１に示すような感じになっています。

　★表１．【ヘッダファイルの内容(例)】
    (ABCD.BAS と ABCD.BA0 の２つのファイルを送る場合)
     2[CR][LF]              →　ファイル数
    ABCD.BAS[CR][LF]        →　ファイル名
    ABCD.BA0[CR][LF]        →　ファイル名
    [EOF]

　　　[CR] は、0Dh コード、[LF] は、0Ah コード、[EOF] は、1Ah コード。


【３】ファイル実行の引き数について

　　　　ファイルを実行するときに、ファイルの後に引き数を付けて実行する
　　　ことができます。
　　　　通常のＢＡＳＩＣプログラムでは意味がありませんが、何か使う場合が
　　　あるかも知れないので、作ってみました。

　　　　引き数の取り込み方は簡単です。

　　　　ＦＴは実行プログラムを内部でCHAINしています。このときに、変数は
　　　受け渡されます。ＦＴがCHAIN時に受け渡す変数は、２つです。
　　　　１つは、Ａ＄で、もう１つはＺＦＴ＄です。

　　　・Ａ＄には、CHAINするファイル名、が収納されています。
　　　・ＺＦＴ＄には、引き数として、ファイル名の後ろに付けられた内容が
　　　　格納されています。

　　　　ベーシックプログラムで、引き数を参照する場合には、このＺＦＴ＄を
　　　読めばＯＫです。


【４】ファイル名の格納ファイルについて

　　　　ＦＴは、読みとったファイル名を ZFT.BA3 ファイルに格納しています。
　　　２回目以降の起動は、全てこのファイルを読み込んで、画面にファイル名
　　　を表示します。
　　　　ZFT.BA3 の更新は、ＦＴの画面の下のクリップマークがタッチされたと
　　　きのみです。

　　　　ファイルの内容は、
　　　ファイル名　→　８バイト
　　　　　．　　　→　１バイト
　　　拡張子　　　→　３バイト
　　　サイズ　　　→　６バイト

　　　の、１ファイルにつき計１８バイトが、つらつら、列挙されて格納されて
　　　いるだけです。改行等はありません。
　　　　そして、行末に &1A コードが入っています。

　　　　例えば、格納ファイルの総数を知りたければ、

　　　　　総数＝（ZFT.BA3 のファイルサイズ−１）／１８
　　　で、求めることができます。
